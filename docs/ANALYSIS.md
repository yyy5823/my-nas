# MyNAS 项目需求分析文档

## 1. 项目概述

MyNAS 是一款跨平台的家用 NAS 连接工具，旨在为用户提供统一、美观、现代化的 NAS 资源访问和管理体验。

### 1.1 目标平台
- **桌面端**: macOS, Windows
- **移动端**: iOS, Android

### 1.2 核心目标
- 提供统一的多端体验
- 完全解耦的模块化架构
- 美观大气的现代化 UI
- 灵活适配不同 NAS 系统

---

## 2. 功能需求分析

### 2.1 媒体播放模块

#### 2.1.1 视频播放
**支持格式:**
- 容器格式: MP4, MKV, AVI, MOV, WMV, FLV, WebM, RMVB, TS, M2TS
- 视频编码: H.264, H.265/HEVC, VP9, AV1, MPEG-4
- 字幕格式: SRT, ASS/SSA, VTT, SUB, PGS, embedded

**功能需求:**
- 在线流媒体播放（支持转码/直接播放）
- 进度记忆与多设备同步
- 字幕加载与样式自定义
- 倍速播放 (0.5x - 3x)
- 画中画模式
- 投屏支持 (AirPlay, DLNA, Chromecast)
- 视频库管理（分类、收藏、观看历史）

#### 2.1.2 音乐播放
**支持格式:**
- 有损: MP3, AAC, OGG, WMA, M4A
- 无损: FLAC, APE, WAV, AIFF, ALAC, DSD

**功能需求:**
- 音乐库浏览（艺术家、专辑、歌曲、播放列表）
- 歌词显示（LRC 逐行歌词、内嵌歌词）
- 均衡器
- 无缝播放/交叉淡入淡出
- 后台播放
- 播放队列管理
- 专辑封面显示

#### 2.1.3 漫画阅读
**支持格式:**
- 图片: JPG, PNG, WebP, GIF, BMP
- 压缩包: ZIP, RAR, 7Z, CBZ, CBR
- 电子书: PDF (漫画模式)

**功能需求:**
- 多种阅读模式（单页、双页、长条）
- 阅读方向设置（从左到右、从右到左）
- 手势翻页与缩放
- 阅读进度记忆
- 漫画库管理（系列、作者、标签）
- 智能预加载

#### 2.1.4 书籍阅读
**支持格式:**
- 电子书: EPUB, PDF, MOBI, AZW3, TXT, FB2
- 文档: DOC, DOCX, RTF

**功能需求:**
- 自定义字体、字号、行距
- 主题切换（日间、夜间、护眼）
- 书签与笔记
- 目录导航
- 全文搜索
- 阅读进度同步
- 文本高亮与批注

### 2.2 笔记模块

**功能需求:**
- Markdown 编辑器
- 富文本编辑
- 笔记分类与标签
- 全文搜索
- 附件管理（图片、文件）
- 云端同步（存储在 NAS）
- 离线编辑与冲突处理
- 导出功能（PDF, HTML, Word）

### 2.3 NAS 系统适配

#### 2.3.1 群晖 (Synology)
**连接方式:**
- WebAPI (DSM 6.x / 7.x)
- QuickConnect
- DDNS/直连

**支持功能:**
- File Station API
- Video Station API
- Audio Station API
- Photos API
- 用户认证与权限

#### 2.3.2 绿联 (UGREEN)
**连接方式:**
- 官方 API（需调研）
- WebDAV
- SMB/CIFS

**支持功能:**
- 文件浏览与管理
- 媒体流服务
- 用户认证

#### 2.3.3 通用协议支持（扩展性）
- WebDAV
- SMB/CIFS
- FTP/SFTP
- NFS (Linux/Mac)

### 2.4 下载工具管理模块

#### 2.4.1 NASTools 集成
- 任务状态查看
- 订阅管理
- 搜索与下载
- 媒体库管理
- 配置管理

#### 2.4.2 qBittorrent 集成
- 下载任务管理
- 种子添加（磁力链接、种子文件）
- 下载进度监控
- 速度限制设置
- 分类管理

#### 2.4.3 其他工具扩展（预留）
- Transmission
- Aria2
- Jackett/Prowlarr
- Radarr/Sonarr

## 3. 非功能需求

### 3.1 性能要求
- 应用启动时间 < 3秒
- 视频播放延迟 < 1秒
- 流畅的 60fps UI 动画
- 内存占用优化

### 3.2 安全要求
- HTTPS 加密通信
- 本地数据加密存储
- 生物识别/密码保护
- 敏感信息安全存储

### 3.3 用户体验
- 响应式设计
- 暗色/亮色主题
- 无障碍支持
- 多语言支持（中/英）
- 手势操作优化

### 3.4 离线能力
- 离线缓存播放
- 笔记离线编辑
- 断点续传

## 4. 技术约束

### 4.1 跨平台方案选择

| 方案 | 优点 | 缺点 | 适用性 |
|------|------|------|--------|
| **Flutter** | 高性能、一套代码、美观UI | 原生能力需桥接 | ⭐⭐⭐⭐⭐ |
| React Native | 生态成熟、热更新 | 性能一般、UI 一致性差 | ⭐⭐⭐ |
| Kotlin Multiplatform | 原生体验、代码复用 | 成熟度不足、学习曲线 | ⭐⭐⭐ |
| Tauri | 轻量、桌面端优秀 | 移动端支持不成熟 | ⭐⭐ |

**推荐方案: Flutter**

### 4.2 架构要求
- 模块化设计
- 依赖注入
- 响应式状态管理
- 清晰的分层架构

## 5. 目标用户画像

### 5.1 主要用户
- 家庭 NAS 用户
- 数字媒体爱好者
- 技术爱好者

### 5.2 用户场景
1. **家庭影院**: 在电视/平板上浏览和播放 NAS 中的电影
2. **移动阅读**: 在手机上阅读电子书和漫画
3. **音乐欣赏**: 随时随地播放 NAS 中的音乐库
4. **远程管理**: 在外出时管理下载任务
5. **知识管理**: 记录和整理个人笔记

## 6. 竞品分析

| 应用 | 优点 | 缺点 |
|------|------|------|
| DS Video/Audio/Photo | 官方支持、稳定 | 功能分散、UI 老旧 |
| Infuse | UI 精美、格式支持全 | 仅视频、价格高 |
| VLC | 开源免费、格式全 | UI 简陋 |
| Plex | 生态完善 | 需要服务端、订阅制 |

**差异化定位:**
- 一站式多媒体解决方案
- 现代化 UI 设计
- 开放的 NAS 适配架构
- 本地化下载工具管理
