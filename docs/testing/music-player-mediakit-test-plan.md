# 音乐播放器 MediaKit 引擎测试方案

## 概述

本文档描述音乐播放器从 `just_audio` 迁移到 `media_kit` 引擎的测试方案。此次迁移旨在支持 AC3、DTS、Dolby TrueHD 等高级音频格式，同时保留原有的平台原生引擎作为备选。

### 相关文件

**核心实现**:
- `lib/features/music/data/services/music_audio_handler_interface.dart` - 统一接口定义
- `lib/features/music/data/services/music_audio_handler.dart` - just_audio 引擎实现
- `lib/features/music/data/services/music_media_kit_handler.dart` - media_kit 引擎实现
- `lib/features/music/data/services/music_audio_passthrough_service.dart` - 音频直通配置

**Provider 层**:
- `lib/features/music/presentation/providers/music_player_provider.dart` - 播放器状态管理
- `lib/features/music/presentation/providers/music_settings_provider.dart` - 设置管理

**UI 层**:
- `lib/features/music/presentation/widgets/music_settings_sheet.dart` - 设置面板（含引擎切换）

**初始化**:
- `lib/main.dart` - 应用启动时根据设置初始化对应引擎

### 设计文档

- `docs/design/music-player-mediakit-migration.md` - 迁移设计方案

## 测试用例

### 第一部分：引擎切换功能测试

#### TC-001: 默认引擎验证

**目的**: 验证应用默认使用 just_audio 引擎

**前置条件**:
- 全新安装应用或清除应用数据

**测试步骤**:
1. 启动应用
2. 进入音乐模块
3. 打开播放设置面板
4. 查看"播放引擎"区块

**预期结果**:
- [ ] 默认选中"平台原生"选项
- [ ] 提示文字显示"当前使用平台原生引擎，更省电但不支持 AC3/DTS 等格式"
- [ ] 日志显示 `初始化音频处理器: just_audio 引擎 (平台原生解码器)`

**验证方法**:
```bash
# 查看应用日志
flutter logs | grep "初始化音频处理器"
```

---

#### TC-002: 切换到 MediaKit 引擎

**目的**: 验证切换引擎流程和持久化

**前置条件**:
- 当前使用 just_audio 引擎

**测试步骤**:
1. 打开播放设置面板
2. 点击"FFmpeg"引擎选项
3. 在弹出对话框中点击"确认切换"
4. 观察 SnackBar 提示
5. 完全退出应用（杀进程）
6. 重新启动应用

**预期结果**:
- [ ] 弹出确认对话框，内容包含"支持 AC3、DTS、Dolby TrueHD"
- [ ] 确认后显示 SnackBar "播放引擎已更改，请重启应用生效"
- [ ] 重启后"FFmpeg"选项被选中
- [ ] 提示文字显示"当前使用 FFmpeg 引擎，支持 AC3、DTS、Dolby 等高级音频格式"
- [ ] 日志显示 `初始化音频处理器: media_kit 引擎 (支持 AC3/DTS/Dolby)`

## 附录

### A. 测试音频文件来源

1. **常规格式**: 使用 FFmpeg 从无损源转换
   ```bash
   ffmpeg -i source.wav -b:a 320k test_mp3.mp3
   ffmpeg -i source.wav test_flac.flac
   ```

2. **高级格式**: 从蓝光/DVD 提取或使用测试音频
   ```bash
   ffmpeg -i source.mkv -c:a copy -vn test_ac3.ac3
   ```

### D. 已知限制

1. **just_audio 引擎**:
   - 不支持 AC3、DTS、Dolby TrueHD 等高级格式
   - 依赖平台原生解码器

2. **media_kit 引擎**:
   - 包体积增加（含 FFmpeg 库）
   - 首次加载可能稍慢
   - 某些平台的系统集成可能有差异

3. **音频直通**:
   - 需要外部设备支持
   - AirPlay 可能不支持直通所有格式
